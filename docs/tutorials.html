Getting Started with Scripting

Beginner

Welcome to BlockVerse Scripting! BlockVerse lets you add interactivity to your games using two methods:

1. Visual Blocks (Beginner Mode) — Drag and snap blocks together
2. JavaScript Code (Advanced Mode) — Write code directly

This tutorial covers the Visual Blocks system. No coding experience needed!

💡
Visual Blocks work like puzzle pieces — each block has a unique shape that only connects to compatible blocks. This prevents errors and makes learning fun!
1

Open the Creator

  • Click "Creator Studio" in the sidebar
  • Wait for the 3D viewport to load
2

Open the Scripting Panel

  • Click the "Scripting" tab on the right panel
  • Make sure "Beginner Mode" is selected (toggle at top)
3

Add Your First Block

  • From the palette on the left, find "Events" (green blocks)
  • Drag "when game starts" to the workspace
  • This is the starting point for your script
4

Add an Action

  • Find "Looks" (purple blocks)
  • Drag "show hint 'Hello World!'" and snap it below the green event block
  • The blocks should click together like puzzle pieces
5

Test It!

  • Click the "Play" button to test your game
  • When the game starts, you should see "Hello World!" on screen
Visual Blocks 
when game starts
  show hint "Hello World!"
JavaScript Equivalent 
// The same script in JavaScript (Advanced Mode)
Events.on('gameStart', function() {
    UI.showHint('Hello World!');
});
🎉
Congratulations! You've created your first script! The event block (green) triggers when the game starts, and the action block (purple) shows a message on screen.

Block Basics

Beginner

Blocks are the building blocks of your world (pun intended!). You can place, remove, and detect blocks using scripts. 

   World Grid (Top-Down View)

        X →
    Z   ┌───┬───┬───┬───┬───┐
    ↓   │   │ G │   │   │   │
        ├───┼───┼───┼───┼───┤
        │   │ G │ G │   │   │
        ├───┼───┼───┼───┼───┤
        │   │ P │ G │ S │   │
        └───┴───┴───┴───┴───┘
              P = Player
              G = Grass
              S = Stone

Placing Blocks

Visual Blocks 
place block at (5, 1, 0) type [grass]
JavaScript 
Block.place(5, 1, 0, 'grass');

Removing Blocks

Visual Blocks 
remove block at (5, 1, 0)
JavaScript 
Block.remove(5, 1, 0);

Checking if a Block Exists

Visual Blocks 
if block at (5, 1, 0) is [grass] then
  show hint "Found grass!"
end
JavaScript 
if (Block.get(5, 1, 0)) {
    UI.showHint("Found grass!");
}

Finding All Blocks of a Type

Visual Blocks 
find all [gold] blocks
JavaScript 
const goldBlocks = Block.find('gold');
// Returns an array of {x, y, z} positions

Player Control

Beginner

Learn how to control the player character — move them around, detect touches, and handle death/respawn.

Getting Player Position

JavaScript 
const pos = Player.getPosition();
// Returns: { x: 0, y: 5, z: 0 }

Teleporting the Player

Visual Blocks 
move player to (10, 5, 0)
JavaScript 
Player.setPosition(10, 5, 0);

Smooth Movement (Tweening)

Visual Blocks 
tween player to (10, 5, 0) over 2 seconds
JavaScript 
Tween.moveTo('player', 10, 5, 0, 2); // 2 second duration

Changing Speed

JavaScript 
Player.setSpeed(10); // Normal is 8, range: 1-20

Detecting Block Touches

Visual Blocks 
when player touches [gold] then
  show hint "You touched gold!"
end
JavaScript 
Player.onTouched('gold', function() {
    UI.showHint("Touched gold!");
});

Player Death & Respawn

JavaScript 
Player.onDied(function() {
    UI.showHint("You died! Respawning...");
});

Player.onRespawned(function() {
    UI.showHint("Welcome back!");
});
💡
Practice: Create a teleporter! (1) Place an obsidian block at (0, 1, 10), (2) Place another at (20, 1, 10), (3) When player touches the first obsidian, teleport to (20, 2, 10), (4) When player touches the second, teleport back!

Making Your First Game — Obby

Beginner

Let's build a complete obstacle course (obby) from scratch! This walkthrough covers everything from setup to publishing. 

    OBBY COURSE LAYOUT

    START                              FINISH
      │                                    │
      ▼                                    ▼
    ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
    │ G │ ─ │ G │ ─ │ G │ ─ │ G │ ─ │ D │
    └───┘   └───┘   └───┘   └───┘   └───┘
      │       │       │       │
    Gold   Gold   Gold   Diamond
   Check   Check   Check   WIN!
      │       │       │
    ~~~lava~~~lava~~~lava~~~
1

Create a New Game

  • Go to Creator Studio
  • Choose the "Obby Course" template
  • Name your game "My First Obby"
2

Set Up the Script

  • Open the Scripting Panel
  • Switch to "Beginner Mode"
  • Drag "when game starts" to workspace
3

Add a Welcome Message

  • Snap "show hint 'Welcome to My Obby!'" below it
  • Add "wait 3 seconds"
  • Add "show hint 'Reach the gold block to win!'"
4

Create Checkpoints

  • Drag "when player touches [gold] then { ... }"
  • Inside: "show hint 'Checkpoint reached!'"
  • Add: "show score 1"
5

Create a Win Condition

  • Drag "when player touches [diamond] then { ... }"
  • Inside: "show hint 'You Win!'"
  • Add: "show score 10"
6

Build the Course

  • Use Build tool to create platforms
  • Place gold blocks as checkpoints
  • Place a diamond block at the end
  • Add lava below as penalty for falling
7

Test and Publish!

  • Click Play to test
  • Adjust difficulty as needed
  • Click Publish when ready!
Complete JavaScript Script 
// Welcome message
Events.on('gameStart', function() {
    UI.showHint('Welcome to My Obby!', 3);
    Tween.wait(3).then(function() {
        UI.showHint('Reach the diamond block to win!', 4);
    });
});

// Checkpoints (gold blocks)
Player.onTouched('gold', function() {
    UI.showHint('Checkpoint reached!');
    UI.showScore(1);
});

// Win condition (diamond block)
Player.onTouched('diamond', function() {
    UI.showHint('You Win!', 5);
    UI.showScore(10);
    Sound.play('victory');
});
⚠️
Tip: Place lava blocks below the platforms so players fall into them if they miss a jump. This creates the classic obby "penalty" mechanic!

Custom Nodes (My Blocks)

Advanced

Custom nodes let you create your own reusable blocks — just like Scratch's "My Blocks"! Once created, they appear in your palette and can be used just like built-in blocks.

Why Create Custom Nodes?

  • Reuse code without copy-pasting
  • Share with other players
  • Organize complex scripts
  • Build your own library of useful blocks
1

Open My Blocks

  • In the Scripting Panel, click "My Blocks" tab
  • Click "Create New Block"
2

Define Your Block

  • Name: "Teleport to Position"
  • Description: "Teleports the player to X, Y, Z"
  • Add inputs: x (number), y (number), z (number)
  • Output type: Action
3

Write the Logic

  • In Advanced mode, write your code
  • OR use visual blocks to define the behavior
4

Save and Use

  • Click Save
  • Your new block now appears in the "Custom" (teal) category
  • Drag it to your workspace and fill in the inputs!

Example: Score System Node

Custom Node Code 
// Node: "Add to Score"
// Description: "Adds points to the player's score"
// Input: points (number)

var current = parseInt(UI.getScore() || '0');
UI.showScore(current + points);

Example: Reset Game Node

Custom Node Code 
// Node: "Reset Game"
// Description: "Resets player position and score"
// Input: (none)

Player.setPosition(0, 5, 0);
UI.showScore(0);
Chat.send("Game Reset!");

Sharing Custom Nodes

1

Export Your Block

Click "Export" on your custom block to get a share code.

2

Share It

Copy the share code and share it in the Community Hub or with friends.

3

Import

Others paste the code in "Import" to add it to their blocks.

🛑
Rules for Custom Nodes: No infinite loops (detected automatically), no accessing other players' data, maximum 50 custom nodes per player, names must be 3-30 characters, no inappropriate content (moderated).

Variables & Game State

Medium

Variables let you store information like scores, timers, and game state. This is essential for creating real games!

📚
What is a variable? Think of it as a labeled box where you can store a value. You can put things in, take things out, and change the value at any time during your game.

Creating Variables

Visual Blocks 
set [myScore] to 0
set [playerName] to "Hero"
set [gameStarted] to true
JavaScript 
var myScore = 0;
var playerName = "Hero";
var gameStarted = true;

Using Variables with Conditions

JavaScript 
var score = 0;

Player.onTouched('gold', function() {
    score += 10;
    UI.showHint("Score: " + score);
    UI.showScore(score);

    if (score >= 50) {
        UI.showHint("You're on fire!");
    }
});

Countdown Timer Example

JavaScript 
var timeLeft = 60; // 60 seconds

UI.showTimer(timeLeft);

Timer.every(1, function() {
    timeLeft -= 1;
    UI.showTimer(timeLeft);

    if (timeLeft <= 0) {
        UI.showHint("Time's up!");
        UI.hideTimer();
        // Reset game or show results
    }
});

Multiplayer Game State

JavaScript 
// Track which players have finished
var finished = {};

Events.on('playerJoin', function(player) {
    finished[player.name] = false;
});

Player.onTouched('diamond', function() {
    finished[Player.getName()] = true;
    Chat.send(Player.getName() + " finished!");
});
💡
Tip: Use var for simple variables. For data that needs to persist across sessions, use the BlockVerse save system: Save.set(key, value) and Save.get(key).
07

Sharing Your Creations

Beginner

Ready to show off your game to the world? Here's how to publish to the Community Hub and share with friends.

1

Test Your Game Thoroughly

Before publishing, play through your entire game. Check for bugs, test edge cases, and make sure the experience is smooth.

2

Write a Great Description

Your description is the first thing players see. Include what makes your game special, controls, and any tips.

3

Publish to Community Hub

  • In Creator Studio, click "Publish"
  • Choose your game's category
  • Set max players per server
  • Click "Publish" to go live!
4

Share with Friends

Every published game gets a Game Code (e.g., BV-A3X7). Share this code so friends can join directly!

5

Get Feedback & Iterate

Watch the play count and listen to player feedback. Update your game regularly to keep players coming back!

🌟
Pro Tip: Games with good thumbnails and descriptions get 3x more plays! Take time to make your game page look attractive.

JavaScript API Reference

Ref

Complete reference for all BlockVerse scripting APIs. Use these functions in Advanced Mode (JavaScript) to create powerful game logic.

🧩 Block API

MethodDescriptionReturns
Block.place(x, y, z, type) Place a block at the given coordinates void
Block.remove(x, y, z) Remove a block at the given coordinates boolean
Block.get(x, y, z) Get block info at coordinates, or null if empty object | null
Block.find(type) Find all blocks of a given type array
Block.onPlaced(callback) Event: fires when any block is placed void
Block.onRemoved(callback) Event: fires when any block is removed void

🏃 Player API

MethodDescriptionReturns
Player.getPosition() Get current player position {x, y, z}
Player.setPosition(x, y, z) Teleport player to coordinates void
Player.getSpeed() Get current movement speed number
Player.setSpeed(speed) Set movement speed (1-20, default 8) void
Player.onTouched(type, cb) When player touches a block type void
Player.onDied(callback) When player falls into void void
Player.onRespawned(callback) When player respawns void
Player.getName() Get the player's username string

💬 Chat API

MethodDescriptionReturns
Chat.send(message) Send message to chat void
Chat.onMessage(callback) When any message is received void

🎨 UI API

MethodDescriptionReturns
UI.showHint(text, duration?) Show hint on screen (default 3 sec) void
UI.showScore(value) Show score on HUD void
UI.getScore() Get current displayed score string
UI.showTimer(seconds) Show countdown timer on HUD void
UI.hideTimer() Hide the countdown timer void
UI.showLeaderboard(players) Show top players void

🎨 Tween API

MethodDescriptionReturns
Tween.moveTo(target, x, y, z, dur) Smooth movement over duration (seconds) Promise
Tween.rotateTo(target, rx, ry, rz, dur) Smooth rotation Promise
Tween.scaleTo(target, sx, sy, sz, dur) Smooth scaling Promise
Tween.wait(seconds) Wait (returns a promise) Promise

🔊 Sound API

MethodDescriptionReturns
Sound.play(name) Play a built-in sound (e.g., 'click', 'victory') void
Sound.stopAll() Stop all sounds void
Sound.setVolume(0-1) Set master volume (0 to 1) void

⚡ Events API

MethodDescriptionReturns
Events.on(event, callback) Listen for an event void
Events.off(event, callback) Remove event listener void
Events.emit(event, data) Emit a custom event void
📚
Built-in events: 'gameStart', 'playerJoin', 'playerLeave', 'blockPlace', 'blockRemove'

⏰ Timer API

MethodDescriptionReturns
Timer.every(seconds, callback) Repeating timer (fires every N seconds) timerId
Timer.after(seconds, callback) One-shot timer (fires once after N seconds) timerId
Timer.stop(id) Stop a running timer void

🔢 Math Helpers

MethodDescriptionReturns
Math.random() Random number between 0 and 1 number
Math.floor(x) Round down to nearest integer number
Math.ceil(x) Round up to nearest integer number
Math.abs(x) Absolute value number
Math.sqrt(x) Square root number
Math.min(a, b, ...) Smallest value number
Math.max(a, b, ...) Largest value number
